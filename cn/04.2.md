# 04.2 _id原理详细分析

## 介绍
MongoDB中存储的文档必须有一个"_id"键。这个键的值可以是任何类型的，默认是个ObjectId对象。 
在一个集合里面，每个集合都有唯一的"_id"值，来确保集合里面每个文档都能被唯一标识。
如果插入文档的时候没有"_id"键，系统会帮你自动创建一个

## 生成规则
```
ObjectId("5875e473947bc25cc4cb6174")   生成规则的解释。。十六进制的数据
4byte：  5875e473   =>   十六进制转换成十进制 1484121203， 然后时间戳装换 2017/1/11 15:53:23
3byte:   3947bc     =>   机器的唯一标识码
2byte：  25cc       =>   进程ID
3byte：  b6174      =>   进程内的随机数
```
